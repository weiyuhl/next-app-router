## 🏗️ 核心设计理念

### 充分利用 Next.js 15 App Router 特性

本方案的最大特点是**充分利用**了 Next.js 15 App Router 的三大核心布局特性：

#### 1️⃣ 路由组（Route Groups）

```
用途：前后台分离、权限控制
示例：(frontend) / (admin) / (auth)
优势：不同区域独立布局，URL保持简洁
```

#### 2️⃣ 嵌套布局（Nested Layouts）

```
用途：模块内二级导航
示例：内容管理、用户管理
优势：避免重复渲染，保持状态
```

#### 3️⃣ 并行路由（Parallel Routes）

```
用途：仪表盘、数据展示
示例：@stats / @charts / @activities
优势：独立加载，支持复杂布局
```

```
咏雪轩阁项目
    │
    ├─ (frontend) 路由组 ─────→ 前台公开访问区

    │
    ├─ (admin) 路由组 ─────────→ 后台管理区
    │   ├─ 使用专业的管理布局
    │   ├─ dashboard (并行路由)
    │   ├─ content (嵌套布局)
    │   ├─ users (嵌套布局)
    │   ├─ analytics (并行路由)
    │   └─ settings (并行路由)
    │
    └─ (auth) 路由组 ──────────→ 认证页面区
        └─ 使用简洁的认证布局
```

## 📦 技术栈

### 前端
- **框架**: Next.js 15.5.0 (App Router)
- **语言**: TypeScript 5
- **样式**: Tailwind CSS v4
- **UI组件**: shadcn/ui
- **图标**: Lucide React
- **图表**: Recharts
- **表单**: React Hook Form + Zod
- **状态管理**: Zustand

### 后端（建议）
- **API**: Next.js Route Handlers
- **数据库**: PostgreSQL / MySQL
- **ORM**: Prisma
- **认证**: NextAuth.js / Clerk
- **文件存储**: AWS S3 / Cloudinary
